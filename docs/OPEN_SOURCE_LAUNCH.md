# Open Source Launch Checklist

## Pre-Launch

- [ ] Confirm license and ownership rights for all included assets/data
- [ ] Ensure no secrets/credentials in template examples
- [ ] Verify README commands from a clean clone
- [ ] Tag current status clearly as `alpha`
- [ ] Add screenshots/GIFs to README

## Launch Day

- [ ] Publish repository as public
- [ ] Add short demo video/GIF
- [ ] Post launch note:
  - LinkedIn
  - X
  - Dev.to / Medium
  - relevant Reddit/HN threads
- [ ] Pin a “Getting Started” discussion/issue

## First 30 Days

- [ ] Track onboarding friction (install, first render, first edit)
- [ ] Prioritize top 3 recurring issues
- [ ] Ship at least 2 incremental releases
- [ ] Publish a transparent changelog
